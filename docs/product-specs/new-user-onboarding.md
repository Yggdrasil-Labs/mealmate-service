# 新用户接入与首次建档

## 背景

MealMate 的价值建立在“家庭画像足够完整”之上。如果首次进入系统的用户无法快速建立家庭成员与基本饮食约束，后续菜品筛选、周计划生成和营养分析都会失真。

## 目标

让新用户在最短路径内完成家庭级基础建档，为后续计划生成与执行提供最小可用数据。

## 主要用户

- 家庭主要维护者
- 初次使用系统、需要建立家庭饮食档案的用户

## 范围内能力

- 创建家庭基础信息
- 添加家庭成员
- 填写成员基础属性、口味偏好、忌口和饮食目标
- 标记宝宝等特殊人群
- 完成首次建档后的状态确认

## 非目标

- 首次接入阶段不生成完整周计划
- 不在此阶段要求完善完整菜品知识库
- 不在此阶段引入 AI 自动推荐替代必填规则

## 关键业务规则

- 家庭至少需要一个成员才能完成首次建档
- 宝宝成员必须具备额外的饮食约束信息
- 忌口、减脂目标、适配人群等约束必须结构化表达
- 首次建档数据必须能直接作为后续周计划生成输入

## 验收标准

- 用户可以完成家庭及成员的最小建档
- 后端能输出结构化家庭画像
- 建档结果满足后续周计划和推荐链路的最小输入要求
