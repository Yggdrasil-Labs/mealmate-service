# 核心信念

## 1. 仓库是长期系统记录

- 对 AI 决策重要的事实、规则和设计结论应进入仓库。
- 聊天、口头同步和临时评论不能替代正式文档。
- 同一件事只能有一个主入口，避免平行真相来源。

## 2. COLA 5.0 分层优先于编码速度

- `adapter` 负责适配，不承载业务规则。
- `app` 负责编排，不替代领域建模。
- `domain` 是业务规则中心。
- `infrastructure` 负责技术实现，不反向污染领域。

## 3. MealMate 不是菜谱展示系统

MealMate 要优化的是“家庭饮食规划与执行”的完整闭环，而不是单纯展示菜谱列表。因此任何实现都应优先服务以下能力：

- 家庭成员差异化饮食约束
- 周计划生成与调整
- 采购与备菜执行
- 实际记录
- 营养复盘

## 4. 文档先于批量实现

- 在大改动、跨模块需求或重要能力建设前，先形成产品规格或设计文档。
- 需求、设计、计划和实现应形成链路，而不是只留下代码结果。

## 5. 优先最小可验证改动

- 倾向于小步推进、快速验证、再扩展。
- 不为了“看起来完整”而引入当前阶段用不到的抽象。

## 6. 质量是多维约束

质量不只等于“代码能跑”，还包括：

- 是否遵守分层边界
- 是否易于 AI 和人类理解
- 是否可验证
- 是否可演进
- 是否对安全和稳定性负责
